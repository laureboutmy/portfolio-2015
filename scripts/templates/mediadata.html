The assignment for this project was to create a datavizualization with the theme «culture & medias» in mind. It was made in May 2014 and I was Product Designer as well as the lead Front-end Developer.

We decided to create a tool using a set of data from the French National Audiovisual Institute. Simply put, the Mediadata app allows you to dissect and compare the media coverage of famous personnalities over 20 years of data and 75 TV/radio channels.

Concept Coming up with an idea thanks to data

During a brainstorming session, we stumbled upon the data library of the French National Audiovisual Institute (INA). It is a repository of all French radio and television audiovisual archives: since 1992, each television and radio channel must record what show is airing, indicating who was present, and what the program was about.

Insight

From then on our problematic was pretty obvious.

Medias have a big influence on us but we are uncertain of their impartiality.

To explain further, our goal was to create a tool that would help people have an oversight of what is being talked about on medias, forcing journalists on more transparency.

Functionalities Designing a useable tool

The biggest challenge in every datavisualisation project is the user experience. The goal is to help our user make sense of the data and get meaning from it.

After dissecting the data made available for us by the INA, we chose to concentrate on personalities exposure and comparison. A few factors helped making this decision: our time was limited, and an API was not available: our Back-end Developer had to crawl the INA website with a PHP script to retrieve all the information we needed. It was easier to focus on people than on topics.

In the end, there are three main features for the Mediadata app: being able to explore, compare and filter.

The explorer

The user does not usually know what to look for when using the app for the first time. That is why we created both a search page with suggestions as well as a library listing all of the available personnalities.

The comparator

Our main concern with this feature was to guide as much as possible our user when comparing two personalities. We decided to fake splitting the screen in two parts: each personalities has one side and one color. For some of the modules that take all of the window’s width, a color code included in the top bar helps you understand the graphs quickly.

Filters

Without the filters, the app does not have much use. They are the feature that allows you to dissect the data. For example, if you are looking to compare candidates to the presidency’ exposure, it only makes sense to do it within the campaign’s dates.

Modules

To display the data, we created multiple modules :

  • A module displaying the data over time
  • A module showing which days of the week and hours where the person is most talked about
  • A module displaying the most popular programs
  • One for the most popular channels
  • A module showing which personalities are they seen/talked of most with

To create those modules, we found our ideas by exploring dribbble and by doing a lot of sketching. The biggest difficulty we encountered was designing without the actual datas: if you are not careful, you can get some weird surprises; We also had to make sur those modules coulb berolled out to comparison.

Future outlook Mediadata 2.0

We believe there really is a purpose to this project. The next step would be to add more personnalities to the app, as well as topics and to create new modules. The only thing standing in the way is the data source: there is no INA API and crawling the data takes quite a long time (François Hollande, for example, took us 2 days to retrieve)!
